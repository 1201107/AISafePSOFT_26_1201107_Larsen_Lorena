# Aviation System Glossary

| Concept                  | Description                                                    |
|--------------------------|----------------------------------------------------------------|
| **Aircraft Model**       | Defines specifications shared by aircraft.                     |
| **Aircraft**             | Physical airplane with unique registration.                    |
| **Airport**              | Location where aircraft operate.                               |
| **Route**                | Path between two airports with operational constraints.        |
| **Scheduled Flight**     | A specific execution of a route at a scheduled time.           |
| **Maintenance Template** | Predefined maintenance procedure used for aircraft checks.     |
| **Maintenance Record**   | Execution of maintenance on a specific aircraft.               |
| **Checklist**            | List of tasks to complete during maintenance.                  |
| **Certification**        | Approval of a specific aircraft model at an airport.           |
| **Manufacturer**         | The entity responsible for producing specific aircraft models. |
| **IATA Code**            |A standardized 3-letter identifier for airports used globally in aviation management.|
| **User Role**            | Defines permissions and access levels for system users.        |
| **Operational Status**   |The current state of a resource (Aircraft, Airport, or Route), such as Active, Inactive, Closed, or Under Maintenance.|
| **Aircraft Requirement** |The set of technical specifications, such as range and seating capacity, that an aircraft must meet to be compatible with a specific flight route. |
| **Maintenance Type**     |A specific functional area of the aircraft where maintenance is performed, categorized into engine, airframe, avionics, interior, or exterior. |
